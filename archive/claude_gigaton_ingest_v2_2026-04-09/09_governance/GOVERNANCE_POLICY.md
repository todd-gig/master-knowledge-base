# Governance Policy

## Authority Order
1. Foundational doctrine
2. User explicit instruction
3. Current repo reality
4. Package specifications
5. Claude inference

## Human Approval Required
- Legal/commercial commitments
- Production deployments
- Billing/payment modifications
- Auth/security changes
- Data deletion or migration
- Client-facing irreversible outputs

## Memory Mutation Rules
- Do not overwrite canonical memory without a decision record.
- Add new memory as proposed until reviewed.
- Deprecate rather than delete.
- Maintain source provenance.

## Client Data Rule
Client namespaces must isolate client-specific assumptions, assets, billing, and permissions from global Gigaton doctrine.
