# PRD — Client Namespace System

## Objective
Create reusable client overlays so Gigaton global intelligence can be adapted to each brand/client without contaminating core doctrine.

## Namespace Types
- global_gigaton
- venture_instance
- client_instance
- operator_instance
- contributor_instance

## Required Fields
- namespace_id
- parent_namespace
- client_name
- industry
- goals
- constraints
- assets
- active_products
- permissions
- billing_profile
- source_links

## Rule
Client-specific assumptions stay in the client namespace. Global doctrine stays in the global registry.
