# What This Repository Is

**master-knowledge-base** is the organizational intelligence repository. It is not a runnable application. It contains:

1. **Claude operating systems** — versioned CLAUDE.md system prompts governing how Claude reasons, routes, and executes across all projects
2. **Decision execution engine** — 22-document framework governing hybrid Claude/Python architecture decisions
3. **Brand & training materials** — brand voice library, skills matrix, Claude training artifacts
4. **Knowledge extracts** — structured Markdown files extracted from xlsx workbooks (auto-generated by `scripts/xlsx_to_knowledge.py`)
5. **Conversation archives** — ChatGPT and Claude thread exports organized by project
6. **For-the-group** — cross-project frameworks: OVS, 3-Layer Memory Architecture, Pascal Protocol, patent IP materials

# Active Claude OS Version: v3.2

The governing intelligence framework is at `claude-systems/v3.2/CLAUDE.md`.

**Core Principle**: Claude discovers logic. Python scales it profitably.

**Routing Decision Tree**:
- `Python-First` — deterministic, high-volume, stable logic (pricing calculations, scoring formulas, DB queries)
- `Claude-First` — ambiguous interpretation, novel situations, exception review, stakeholder communication
- `Hybrid` — Claude parses/scores → Python decides/executes, or Python runs → Claude reviews exceptions

# Decision Execution Engine Framework

# 3-Layer Memory Architecture (Pascal Protocol)

Documented in `for-the-group/March 18, 2026/`. Three layers:
1. **Working Memory** — current conversation context
2. **Session Memory** — thread-level state, persisted across turns
3. **Long-Term Memory** — this knowledge base and structured files

**Bootstrap Protocol**: When starting a new Claude session on a complex project, load the relevant CLAUDE.md + knowledge-extracts + decision-execution-engine docs to restore full context.

# Rules for This Repository

1. **Never put runnable application code here** — this is a knowledge and governance repo.
2. **Version the Claude OS** — new CLAUDE.md versions go in `claude-systems/vX.Y/`, never overwrite.
3. **Knowledge extracts are generated, not hand-edited** — update the xlsx source, then re-run the extractor.
4. **Decision framework docs are additive** — add new docs with next number; never delete existing numbered docs.
5. **Thread exports are immutable** — treat as historical record; add new exports, never edit old ones.
