* * *

1\. Inputs You Must Have (or Fake with Placeholders)
----------------------------------------------------

### A. Core Narrative Inputs (Non-Negotiable)

If these are missing, fabricate v1 placeholders and proceed.

*   **Audience** (exec, buyer, technical, partner)
*   **Primary objective** (sell, educate, onboard, secure next meeting)
*   **Single KPI outcome** (conversion, understanding, commitment, data capture)
*   **Core “Why”** (1 sentence, outcome-driven)
*   **Core “So What”** (economic or strategic impact)

> Decision rule: If it doesn’t move the KPI, it does not go in the deck.

* * *

### B. Content Assets (Minimum Viable)

*   Brand voice reference (or generic corporate-neutral)
*   Product/service one-liner
*   3 value drivers (cost ↓, speed ↑, risk ↓, revenue ↑)
*   1 proof point (case, stat, logic chain)
*   1 CTA (book, calculate, input data, proceed)

* * *

2\. Slide Architecture (Fastest High-Conversion Structure)
----------------------------------------------------------

**Target: 6–8 slides max**

1.  **Hook / Executive Problem**
    *   One headline
    *   One quantified pain
2.  **Current State Failure**
    *   Why status quo breaks at scale
3.  **Future State Vision**
    *   What changes if solved
4.  **Solution Mechanism**
    *   How it works (high level, no internals)
5.  **Value Math**
    *   Simple ROI logic (before → after)
6.  **Interactive Moment**
    *   Input fields / buttons / choices
7.  **Output / Insight Reveal**
    *   Personalized result or scenario
8.  **CTA**
    *   Next step with urgency

* * *

3\. Interaction Design (Keep It Primitive)
------------------------------------------

You are not building a product. You are simulating value.

### Supported Interaction Types (Pick ONE):

*   Button-based navigation
*   Simple form inputs (3–5 fields max)
*   Scenario toggle (A/B/C)
*   Calculator-lite (visible math)

> Rule: Interactivity exists to **extract data or increase belief**, not to impress.

* * *

4\. Tooling Stack (Zero Friction)
---------------------------------

### Fastest Options:

*   **Google Slides + hyperlinks** (fastest)
*   **Figma prototype** (cleaner UX, slightly slower)
*   **Gamma / Tome / Pitch** (acceptable if you already know them)

Avoid:

*   Custom code
*   Over-styled animation
*   New tools you’ve never used

* * *

-----------------------------------------------

*   Write:

    *   KPI
    *   One-sentence “Why”
    *   CTA
*   Decide interaction type

### Minute 5–15: Build Slides

*   Create all slides with **headlines only**
*   No body text yet
*   Ensure logical flow

### Minute 15–22: Add Interaction

*   Insert buttons / links
*   Add form or calculator slide
*   Wire navigation (even if crude)

### Minute 22–27: Populate Value

*   Add:

    *   One ROI number (even estimated)
*   Kill anything non-essential

### Minute 27–30: Sanity Check

*   Does it:
    *   Tell a coherent story?
    *   Ask for something?
    *   Output something personalized?
*   Ship it.

**Liquefex IP Briefing App** is a **narrative-driven, interactive web experience** that replaces static PDFs with:

*   a **guided storyline** optimized for attorney comprehension and time constraints
*   an **on-demand annotation system** (popups) that translates business/technical claims into IP-relevant guidance
*   a **portfolio builder output** (priority claim wedges, next questions, and a suggested filing plan)

1.2 Success Criteria (Business + UX)
------------------------------------

**Success = counsel can exit with:**

*   a coherent understanding of Liquefex OS architecture
*   a clear list of patentable “wedges” and why they are defensible
*   a recommended portfolio strategy (foundational + component + governance + value attribution)
*   a checklist of inputs needed to draft claims and assess prior art risk zones

**UX KPIs**

*   Completion rate of the “IP Counsel Path” ≥ 70%
*   Time-to-clarity: user reaches “Portfolio Map” in ≤ 5 minutes
*   Annotation engagement: ≥ 3 annotations opened on average before output screen
*   Output export: ≥ 50% of users export PDF/brief

2) Audience Model: IP Counsel (Tim Markison-type persona)
=========================================================

2.1 Needs & Constraints
-----------------------

*   Wants **claim-ready clarity**, not marketing.
*   Optimizes for **defensibility**, **portfolio leverage**, and **valuation uplift**.
*   Will probe: novelty boundaries, definitions, enablement, prior art exposure, and scope.

2.2 Design Implications
-----------------------

*   Default to **structured diagrams + definitions + examples**.
*   Keep main narrative clean; push deep IP guidance into **annotation popups**.
*   Provide **evidence base** (source list and “pending transcripts” disclosures).
*   Provide **exportable outputs** (portfolio map + claim seeds + counsel checklist).

4) Core Interaction Model
=========================

4.1 Main Story = “Clean Read”
-----------------------------

Each page delivers:

*   one headline
*   3–6 bullets max
*   one diagram/visual
*   1–2 interactive elements (toggle, “learn more”, or quick question)

4.2 Annotations = “Popup IP Counsel Layer”
------------------------------------------

Annotations are contextual popups attached to highlighted terms or icons.

### Annotation Types

1.  **IP Angle**: “What is protectable here”
2.  **Claim Seed**: pseudo claim language framing
3.  **Defensibility**: why it’s hard to design around
4.  **Prior Art Risk Zone**: where to be cautious / what to search
5.  **Evidence Note**: what source supports this + confidence level
6.  **Trade Secret Flag**: content that should be withheld from broad versions

### Popup Behavior

*   Opens as **side drawer** on desktop; **bottom sheet** on mobile.
*   Includes:
    *   short title
    *   1–3 tight paragraphs
    *   “Add to Portfolio Map” button (saves wedge)
    *   optional “Add counsel question” input

4.3 Micro-Assessments (Attorney Comprehension)
----------------------------------------------

At key points, a 1-question check:

*   “Is this best protected via patent, trade secret, or both?”
*   “Does this rely on unique data structure, workflow, or governance threshold?”

Purpose:

*   reinforces comprehension
*   helps tailor the final portfolio recommendation

------------------------

User clicks “Evidence Base”

*   sees included sources + missing transcripts
*   can request missing evidence or upload docs

function cx(...parts) {
  return parts.filter(Boolean).join(" ");
}

function useLocalStorageState(key, initialValue) {
  const [val, setVal] = useState(() => {
    try {
      const raw = localStorage.getItem(key);
      return raw ? JSON.parse(raw) : initialValue;
    } catch {
      return initialValue;
    }
  });
  useEffect(() => {
    try {
      localStorage.setItem(key, JSON.stringify(val));
    } catch {}
  }, [key, val]);
  return [val, setVal];
}

function downloadJson(filename, obj) {
  const blob = new Blob([JSON.stringify(obj, null, 2)], { type: "application/json" });
  const url = URL.createObjectURL(blob);
  const a = document.createElement("a");
  a.href = url;
  a.download = filename;
  document.body.appendChild(a);
  a.click();
  a.remove();
  URL.revokeObjectURL(url);
}

function clamp(n, min, max) {
  return Math.max(min, Math.min(max, n));
}

            const leverage =
              s.leverage ?? it.sliders.find((x) => x.key === "leverage")?.default ?? 5;

            return (
              <Card key={it.id} title={it.label}>

                    const val = s[sl.key] ?? sl.default ?? 5;

                        </div>

                          step={sl.step}

                          onChange={(e) => {

                              ...prev,
                              [it.id]: { ...(prev[it.id] || {}), [sl.key]: n },
                            }));

                        />
                      </div>

function Pill({ theme, text, variant = "default" }) {
  const bg =
    variant === "accent"
      ? "rgba(106,228,255,0.14)"
      : variant === "muted"
      ? "rgba(255,255,255,0.08)"
      : "rgba(167,139,250,0.12)";
  const border =
    variant === "accent"
      ? "rgba(106,228,255,0.35)"
      : variant === "muted"
      ? "rgba(255,255,255,0.16)"
      : "rgba(167,139,250,0.25)";
  const color =
    variant === "accent" ? theme.accent : variant === "muted" ? theme.muted : theme.accent2;

  return (
    <span
      style={{
        display: "inline-flex",
        alignItems: "center",
        gap: 6,
        padding: "4px 10px",
        borderRadius: 999,
        border: `1px solid ${border}`,
        background: bg,
        color,
        fontSize: 12,
        fontWeight: 800,
      }}
    >
      {text}
    </span>
  );
}

        }, [key, val]);
        return [val, setVal];
      }

      function downloadJson(filename, obj) {
        const blob = new Blob([JSON.stringify(obj, null, 2)], { type: "application/json" });
        const url = URL.createObjectURL(blob);
        const a = document.createElement("a");
        a.href = url;
        a.download = filename;
        document.body.appendChild(a);
        a.click();
        a.remove();
        URL.revokeObjectURL(url);
      }

        const exportPayload = useMemo(() => ({
          app: CONTENT.app.title,
          role,

          savedWedges: savedWedgeList.map(x => ({ ...x.wedge, priority: x.meta.priority, savedAt: x.meta.savedAt })),
          questions,
          evidence: CONTENT.evidence,
        }), [role, savedWedgeList, questions]);

        function goNext() {
          const next = orderedPageIds[pageIndex + 1];
          if (next) setPageId(next);
        }
        function goBack() {
          const prev = orderedPageIds[pageIndex - 1];
          if (prev) setPageId(prev);
        }
        function openAnnotation(id) {
          setActiveAnnotationId(id);
          setDrawerOpen(true);
        }
        function closeDrawer() {
          setDrawerOpen(false);
          setActiveAnnotationId(null);
        }
        function saveWedge(wedgeId, priorityOverride) {
          const w = wedgesById.get(wedgeId);
          if (!w) return;
          setSavedWedges(prev => ({
            ...prev,
            [wedgeId]: {
              wedgeId,
              priority: priorityOverride || prev[wedgeId]?.priority || w.priorityDefault || "P2",
              savedAt: new Date().toISOString(),
            }
          }));
        }
        function unsaveWedge(wedgeId) {
          setSavedWedges(prev => {
            const next = { ...prev };
            delete next[wedgeId];
            return next;
          });
        }
        function addQuestion(text, wedgeId) {
          const t = (text || "").trim();
          if (!t) return;
          setQuestions(prev => ([
            ...prev,
            { id: "q_" + Math.random().toString(36).slice(2,10), text: t, wedgeId: wedgeId || null, pageId, createdAt: new Date().toISOString() }
          ]));
        }
        function resetAll() {
          setRole("IP Counsel");
          setPageId("landing");
          setSavedWedges({});
          setQuestions([]);
          closeDrawer();
        }

        const activeAnnotation = activeAnnotationId ? annotationsById.get(activeAnnotationId) : null;
        const relatedWedge = activeAnnotation?.wedgeId ? wedgesById.get(activeAnnotation.wedgeId) : null;
        const isSaved = activeAnnotation?.wedgeId ? !!savedWedges[activeAnnotation.wedgeId] : false;

        const chapterLabel = page.chapter ? `Chapter ${page.chapter}` : null;

        return (
          <div className="wrap">
            <div className="hdr">
              <div>
                <div className="row">
                  <div className="hdr-title">{CONTENT.app.title}</div>
                  {chapterLabel && <span className="pill purp">{chapterLabel}</span>}
                  <span className="pill accent">{role}</span>
                  {pageId !== "landing" && <span className="pill muted">{progress}%</span>}
                </div>
                <div className="hdr-sub">Prototype • Interactive story + IP annotation layer + portfolio export</div>
              </div>

              <div className="row">
                <button className="btn" onClick={() => setPageId("landing")}>Home</button>
                <button className="btn" onClick={() => setPageId("portfolio")}>Portfolio</button>
                <button className="btn" onClick={() => setPageId("checklist")}>Checklist</button>
                <button className="btn" onClick={() => setPageId("export")}>Export</button>
              </div>
            </div>

            {pageId !== "landing" && (
              <div className="bar">
                <div style={{ width: progress + "%" }} />
              </div>
            )}

            <div className="grid">
              {page.kind === "landing" && (
                <>
                  <div className="hero">
                    <h1>{CONTENT.app.title}</h1>
                    <p>{CONTENT.app.subtitle}</p>

                  <div className="card">
                    <div className="card-title">Select a briefing mode</div>
                    <div className="row">
                      {["IP Counsel","Investor","Operator/Internal"].map(r => (
                        <button key={r} className={"btn " + (role === r ? "primary" : "")} onClick={() => setRole(r)}>
                          {r}
                        </button>
                      ))}
                    </div>
                    <div className="tiny" style={{ marginTop: 10 }}>
                      This prototype is optimized for <b>IP Counsel</b>. Other modes can reuse the same content model.
                    </div>
                    <div className="row" style={{ marginTop: 14 }}>
                      <button className="btn primary" onClick={() => setPageId("hook")}>Start IP Briefing</button>
                      <button className="btn" onClick={() => setPageId("portfolio")}>Jump to Portfolio Map</button>
                      <button className="btn" onClick={() => setPageId("checklist")}>Jump to Counsel Checklist</button>
                    </div>
                  </div>

                  <div className="card">
                    <div className="card-title">Quick controls</div>
                    <div className="row">
                      <button className="btn" onClick={resetAll}>Reset prototype state</button>
                      <button className="btn" onClick={() => downloadJson("liquefex_ip_briefing_export.json", exportPayload)}>
                        Download current export (JSON)
                      </button>

                  </div>

                <>
                  <div className="card">
                    <div className="card-title">Saved wedges</div>
                    {savedWedgeList.length === 0 ? (
                      <div className="tiny">No wedges saved yet. Open annotations during the story and click “Add to Portfolio Map”.</div>

                          <div key={wedge.id} className="card" style={{ background: "rgba(255,255,255,0.02)" }}>
                            <div className="row" style={{ justifyContent: "space-between" }}>
                              <div>
                                <div className="row">
                                  <b>{wedge.name}</b>
                                  <span className="pill purp">{wedge.family}</span>
                                  <span className="pill accent">{meta.priority}</span>
                                </div>
                                <div className="tiny" style={{ marginTop: 6 }}>{wedge.summary}</div>
                              </div>
                              <button className="btn" onClick={() => unsaveWedge(wedge.id)}>Remove</button>
                            </div>

                            <div className="split" style={{ marginTop: 10 }}>
                              <div className="box">
                                <div className="tiny"><b>Claim seed</b></div>
                                <div style={{ marginTop: 6 }}>{wedge.claimSeed}</div>
                              </div>
                              <div className="box">
                                <div className="tiny"><b>Defensibility</b></div>
                                <div style={{ marginTop: 6 }}>{wedge.defensibility}</div>
                                <div className="tiny" style={{ marginTop: 10 }}><b>Prior art risk</b></div>
                                <div style={{ marginTop: 6 }}>{wedge.priorArtRisk}</div>

                            </div>

                            <div className="row" style={{ marginTop: 10 }}>
                              <span className="tiny">Priority:</span>

        return (
          <div style={{ display: "grid", gap: 10, marginTop: 12 }}>

              const wedge = q.wedgeId ? wedgesById.get(q.wedgeId) : null;

                  <div className="row">
                    <span className="pill muted">Page: {q.pageId}</span>
                    {wedge && <span className="pill purp">Wedge: {wedge.family}</span>}
                    <span className="tiny">{new Date(q.createdAt).toLocaleString()}</span>
                  </div>
                  <div style={{ marginTop: 6 }}>{q.text}</div>
                </div>
              );
            })}
          </div>
        );

            </div>
          </div>
