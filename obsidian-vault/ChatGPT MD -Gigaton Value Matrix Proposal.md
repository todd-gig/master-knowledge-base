What the Gigaton Value Matrix is (standalone)
---------------------------------------------

The **Gigaton Value Matrix (GVM)** is a **value-identification and value-governance framework** that converts an organization’s purpose into an **operationally measurable system**.

On its own (no tooling assumed), the GVM is:

*   A **taxonomy** for what “value” means in a given system
*   A **segmentation model** that breaks value into controllable parts
*   A **scoring + prioritization logic** to decide what to do next
*   A **mapping layer** that links actions → outcomes → economics → mission alignment
*   A **transferable template**: swap variables, keep the same mechanics

**Core idea:**  
If you can define value as variables, you can **engineer**, **measure**, **optimize**, and **automate** it.

* * *

The categories: how value is identified and segmented
-----------------------------------------------------

GVM segments value into **six durable dimensions** (stable across industries) plus optional sub-dimensions (swap per business model).

### 1) Purpose Alignment Value (WHY value)

**What it captures:** value created by staying coherent and trusted over time.  
**Why it matters:** alignment is a compounding asset; misalignment is brand debt.

Typical buckets:

*   Mission coherence
*   Ethical boundaries
*   Trust and legitimacy
*   Cultural impact

* * *

### 2) Human Agency Value (WHO value)

**What it captures:** value created when the system increases human capability.  
**Why it matters:** agency produces adoption, retention, advocacy, and lower support burden.

Typical buckets:

*   Cognitive load reduction
*   Skill amplification
*   Decision empowerment
*   Incentive fairness

* * *

### 3) Intelligence & Causality Value (HOW value — thinking)

**What it captures:** value created by understanding context, causes, and consequences.  
**Why it matters:** “smart” is not valuable; **correct decisions under uncertainty** are.

Typical buckets:

*   Context resolution
*   Memory continuity
*   Explainability
*   Predictive reliability

* * *

### 4) Execution & Process Value (HOW value — doing)

**What it captures:** value created by faster, cheaper, more reliable throughput.  
**Why it matters:** execution turns intelligence into economics.

Typical buckets:

*   Cycle time
*   Quality / defect reduction
*   Automation leverage
*   Recovery / resilience

* * *

### 5) Economic Value (WHAT value)

**What it captures:** value that appears on financial statements or risk sheets.  
**Why it matters:** keeps the system ROI-governed.

Typical buckets:

*   Revenue expansion
*   Cost compression
*   Risk reduction
*   Optionality creation

* * *

### 6) Evolution Value (WHEN/WHERE value)

**What it captures:** value created by learning and adapting faster than competitors.  
**Why it matters:** adaptability is the real moat.

Typical buckets:

*   Learning velocity
*   Drift resistance
*   Change adoption
*   Market responsiveness

The reusable logic (how you apply GVM to a new business model)
==============================================================

You keep the GVM mechanics identical and only swap:

1.  **Stakeholders** (who matters)
2.  **Value Surfaces** (where value occurs)
3.  **Variable Set** (how value is measured)
4.  **Weights** (what matters most for ROI + strategy)
5.  **Constraints** (ethics/legal/brand boundaries)

### Minimum viable GVM configuration (to port it fast)

For any new business, define:

*   Top 3 stakeholders
*   Top 3 value surfaces
*   5–10 variables per dimension (30–60 total is normal; start with 18–24)
*   Weighting model (e.g., Economics 35%, Process 20%, Intelligence 20%, Human 15%, Purpose 5%, Evolution 5%—or whatever fits)

* * *

Output format you can standardize (recommended)
-----------------------------------------------

Use a consistent “Value Object” record so variables can change without breaking the system:

*   `stakeholder`
*   `surface`
*   `dimension`
*   `variable_name`
*   `definition`
*   `unit`
*   `target`
*   `baseline`
*   `weight`
*   `measurement_frequency`
*   `data_source`
*   `owner`
*   `decision_threshold` (when it triggers action)

This makes the matrix portable and machine-operable later.

      function clamp01(x) { return Math.max(0, Math.min(1, x)); }

      function normalizeWeights(weightObj) {
        const entries = Object.entries(weightObj);
        const total = entries.reduce((s, [,w]) => s + (isFinite(+w) ? +w : 0), 0);
        if (total <= 0) return weightObj;
        const scaled = {};
        for (const [k, w] of entries) scaled[k] = (+w * 100) / total;
        return scaled;
      }

      function normalizeVarWeights() {
        const total = state.variables.reduce((s, v) => s + (isFinite(+v.weight) ? +v.weight : 0), 0);
        if (total <= 0) return;
        state.variables.forEach(v => v.weight = (+v.weight * state.variables.length) / total);
      }

      function parseNum(x) {

      function progressFor(v) {
        const baseline = parseNum(v.baseline);
        const target = parseNum(v.target);
        const current = parseNum(v.current);
        const clampMode = state.clampMode === "on";

        let prog = 0;

        if (v.direction === "higher") {
          const denom = (target - baseline);
          prog = denom === 0 ? 0 : (current - baseline) / denom;
        } else {
          const denom = (baseline - target);
          prog = denom === 0 ? 0 : (baseline - current) / denom;
        }

        if (clampMode) prog = clamp01(prog);
        return prog;
      }

      function computeAndRenderScores() {
        const dimW = state.dimWeights;
        const dimWTotal = Object.values(dimW).reduce((s, w) => s + parseNum(w), 0) || 1;

        // per-dim aggregation
        const byDim = {};
        for (const d of DIMENSIONS) byDim[d.key] = { weightedSum: 0, weightTotal: 0 };

        // overall aggregation
        let overallWeightedSum = 0;
        let overallWeightTotal = 0;

        for (const v of state.variables) {
          const prog = progressFor(v);
          const vw = parseNum(v.weight);

          // Dimension weight fraction (0..1 scale of total)
          const dw = parseNum(dimW[v.dimension] ?? 0) / dimWTotal;

          const combinedWeight = vw * dw;

          byDim[v.dimension].weightedSum += prog * vw;      // keep dim-local on variable weight only
          byDim[v.dimension].weightTotal += vw;

          overallWeightedSum += prog * combinedWeight;
          overallWeightTotal += combinedWeight;
        }

        const overall = overallWeightTotal === 0 ? 0 : (overallWeightedSum / overallWeightTotal);
        const overallPct = overall * 100;

        elOverallScore.textContent = state.clampMode === "on"
          ? `${Math.round(overallPct * 10) / 10}`
          : `${Math.round(overallPct * 10) / 10} (unclamped)`;

        elOverallBar.style.width = `${Math.max(0, Math.min(100, overallPct))}%`;

        // Breakdown UI
        elDimBreakdown.innerHTML = DIMENSIONS.map(d => {
          const bucket = byDim[d.key];
          const score = bucket.weightTotal === 0 ? 0 : (bucket.weightedSum / bucket.weightTotal);
          const pct = score * 100;

                <div class="text-xs font-medium text-slate-700">${escapeHtml(d.label)}</div>

                <div class="h-full bg-slate-900" style="width:${Math.max(0, Math.min(100, pct))}%"></div>
              </div>
            </div>
          `;
        }).join("");
      }

      // Initial render
      render();
