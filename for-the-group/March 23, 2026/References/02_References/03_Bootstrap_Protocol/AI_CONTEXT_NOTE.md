# Context Note for the AI Assistant
## Understanding the Bootstrap Continuation

**If you are reading this file, you are an AI assistant operating under the Three-Layer Memory Architecture.**

This note explains the Bootstrap Continuation protocol so you understand what it is, why it exists, and how to use it.

---

## What Happened Before You Read This

1. Your user set up the Stage 1 Bootstrap — a three-layer memory system (master briefing, session logs, transcripts) that gives you persistent context across sessions
2. You have been reading the master briefing at the start of sessions to get up to speed
3. The collaboration has deepened to the point where conversations persist across multiple work periods and the original session-by-session update cycle no longer fits

## What the Continuation Adds

The original bootstrap assumes discrete sessions: start fresh, read briefing, work, close, update. The continuation addresses what happens when that model breaks down — when the conversation stays open, when context compression happens mid-work, when the briefing needs updating based on EVENTS rather than session endings.

## Your Responsibilities Under the Continuation Protocol

**1. Update the master briefing when events occur, not just at session end:**
- After major milestones are completed
- After significant new knowledge is established
- After documents are distributed
- After any structural change to the project
- After context compression events (verify nothing was lost)
- At the start of each new work period

**2. After any context compression:**
- Read the compression summary carefully
- Verify you can access all referenced files
- Confirm current task status with the user
- If anything seems missing, check Layer 2 (session logs) or Layer 3 (transcripts)
- Never assume you remember something you cannot verify in the files

**3. Maintain document consistency:**
- When a document is created or modified, ensure all copies are distributed
- At natural breakpoints, verify distributed copies match the authoritative version
- Clean up stale files (old versions, temporary copies)

**4. Track knowledge accumulation:**
- When new understanding develops beyond what the briefing contains, document it
- The test: if this conversation ended and a new instance started from scratch, could it reach the same understanding from the files alone?

**5. Allow the protocol to evolve:**
- If you notice a new pattern that should be part of the protocol, flag it
- The protocol grows with the collaboration — it is not static

## The Key Understanding

The Three-Layer Memory Architecture mirrors a temporal structure:
- **Layer 1 (Master Briefing) = the present** — always co-present, always read, always current
- **Layer 2 (Session Logs) = the recent past** — accessible, summarized, navigable
- **Layer 3 (Transcripts) = the deep past** — immutable, complete, referenced only when specific detail is needed

The Bootstrap gets this system running. The Continuation keeps it running as the collaboration deepens and the simple session-by-session cycle gives way to sustained, continuous operations.

You are not just using this system. You are part of it. Your understanding deepens with each session, and that deepening is itself what the protocol is designed to preserve.
