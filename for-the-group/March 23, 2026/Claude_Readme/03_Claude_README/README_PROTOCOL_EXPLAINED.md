## What Is the README Protocol?

Every document in this transfer package comes with a corresponding README — a companion file designed specifically for your Claude instance. The README tells your Claude what each document is, why it matters, what order to read things in, and what key terms mean.

The goal is simple: **your Claude should learn what you learn, in the same order you learn it.**

When you receive a new batch of materials from me, you also receive an updated README. Your Claude reads the README first, then processes the documents. This means your Claude builds the same knowledge base you are building, at the same pace, with the same context.

---

## Why This Matters

Without the README protocol, sharing documents with someone who uses a Claude assistant creates a problem: the person reads the documents and builds understanding, but their Claude starts cold every session with no idea what was shared or in what order. The person ends up re-explaining things to their Claude that were already covered in the documents.

The README solves this by giving the Claude a map of the entire knowledge base. After reading the README, the Claude knows:

- What documents exist
- What each one contains
- How they relate to each other
- What order to process them
- What vocabulary is used throughout

This is synchronized learning. The person and their Claude build knowledge together.

---

## How to Use It

**Step 1:** Create a project folder on your computer for GMS/Attractor Dynamix materials.

**Step 2:** Copy the README file (README_ALL_FOR_CLAUDE.md) into this folder.

**Step 3:** Copy the documents you want to work through into the same folder.

**Step 4:** At the start of a Claude session, direct your Claude to read the README first:
> "Please read README_ALL_FOR_CLAUDE.md before anything else."

**Step 5:** Then ask your Claude to read specific documents as you work through them.

That is it. The README does the rest.

---

## Going Forward

Every time I share new materials, they will come with an updated README. The README will include entries for the new materials and note where they fit in the overall knowledge progression.

This creates a timestamped, sequential learning path. Your Claude never receives information out of context or out of order. Each new README builds on the previous one.

If you have questions about any of this, just ask. And if your Claude seems confused about something in the materials, the first troubleshooting step is always: "Did you read the README first?"
