## The Three-Layer Memory Architecture

The system uses three layers, each with a distinct purpose and read frequency:

| Layer | File / Source | Purpose | Read Frequency |
|-------|--------------|---------|----------------|
| **Layer 1 — Fixed Past** | MASTER_BRIEFING.md | The single authoritative source of truth — all entities, routing rules, terminology, and conventions. Never grows unbounded. Only architecture-level facts live here. | Every session |
| **Layer 2 — Eternal Now** | Session Logs | Distilled records of what each session accomplished, decided, and deferred. All sessions are equally accessible — none recede into archive. | On demand (most recent) |
| **Layer 3 — Becoming Future** | Full Transcripts | The immutable source archive. Referenced only when a specific past exchange must be verified. Growing, open-ended, never pruned. | Never at session start |

## The Pascal Protocol

The Pascal Protocol is a geometric coordinate system for organizing and cross-referencing project documents. Each document or concept is assigned a pyramid address (e.g., `MS.4.A.2`) that simultaneously encodes:

- Its position in the folder hierarchy
- Its relationship to sibling documents
- Its inheritance from parent concepts

The protocol takes its name from Pascal's Triangle — a structure where each position is the sum of what came before it, and where the same pattern recurs across rows at every scale. Each address inherits the context of its parents. Cross-referencing becomes a matter of citing an address rather than re-explaining a topic.

**Key rule:** When a position accumulates three or more documents, it triggers a sub-pyramid expansion — spawning child addresses. This is the system's natural scaling mechanism.
