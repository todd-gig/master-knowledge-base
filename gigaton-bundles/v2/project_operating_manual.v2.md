# Project Operating Manual v2

## What this bundle is
This bundle is the operating substrate for a Claude project that needs to reason, retrieve, structure outputs, and preserve Gigaton doctrine.

## Core operating model
- Doctrine first
- Then decision systems
- Then execution systems
- Then commercial and platform layers

## Recommended workflow
1. Parse the memory registry JSON.
2. Load all Tier 1 global memories.
3. Identify the user task type.
4. Load relevant Tier 2 and, if necessary, Tier 3 memories.
5. Apply an optional client namespace overlay.
6. Generate outputs using the markdown contract.
7. Preserve IDs and tags for future retrieval.

## When to create a client namespace
Create a client namespace when a client needs:
- different scoring weights,
- different terminology,
- different commercial templates,
- industry-specific data mappings,
- unique workflow variants.

Do not create a client namespace to change:
- foundational values,
- ethics,
- human-centered technology doctrine,
- core doctrine hierarchy.
