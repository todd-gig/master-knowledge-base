# Granular Data Matrix

## Objective
Provide the minimum normalized fields needed to automate qualification, trust certification, and execution.

| Category | Field | Description |
|---|---|---|
| Identity | decision_id | unique record id |
| Identity | initiative_id | parent workstream |
| Intent | problem_statement | what must change |
| Intent | requested_action | proposed move |
| Value | expected_revenue_impact | direct/indirect upside |
| Value | expected_cost_impact | cost increase/decrease |
| Value | time_to_value | how quickly benefit lands |
| Value | compounding_effect | reusable or flywheel effect |
| Risk | downside_magnitude | worst-case consequence |
| Risk | reversibility | rollback feasibility |
| Trust | evidence_quality | source integrity |
| Trust | precedent_strength | prior similar success |
| Trust | auditability | can the chain be reviewed |
| Alignment | doctrine_fit | alignment to core purpose |
| Alignment | ethos_fit | alignment to behavior rules |
| Execution | owner | accountable human/system |
| Execution | approvals_required | needed signoffs |
| Execution | dependencies | blockers and prerequisites |
| Execution | monitoring_metric | success or failure signal |
| Execution | review_date | next control point |
