# First Principles Registry

## Purpose
This registry defines the base logic atoms that must be used when decomposing any strategic or operational decision.

## Core Set
1. **Reality before preference** — decisions must map to observable constraints and incentives.
2. **Value is relational** — value exists relative to a stakeholder, objective, and time horizon.
3. **All resources are finite** — time, capital, attention, trust, and execution bandwidth are constrained.
4. **Speed matters when opportunity decays** — delayed action can destroy value.
5. **Trust is earned through repeated validated outcomes**.
6. **Every system has incentives** — identify them before deciding.
7. **Risk is asymmetrical** — downside and upside are rarely equal.
8. **Compounding dominates isolated wins** — prefer flywheels over isolated gains.
9. **Clarity reduces entropy** — structured language improves execution quality.
10. **Evidence quality determines decision quality ceiling**.
11. **Reversible decisions deserve more speed than irreversible decisions**.
12. **Humans adopt what they understand and trust**.
13. **Goodhart risk applies** — metrics can be gamed if detached from reality.
14. **Ethics are a system constraint, not a branding layer**.
15. **The map is not the territory** — models must be updated from outcomes.

## Application Protocol
For every meaningful decision:
- select 3 to 7 relevant principles,
- state how each principle constrains the decision,
- identify which principle is most load-bearing,
- note any principle conflicts,
- resolve conflicts explicitly.
