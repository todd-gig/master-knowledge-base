# Loop
1. Claude handles ambiguous work.
2. Variables and outcomes are logged.
3. Repeated patterns are identified.
4. Patterns are converted into explicit rules or scoring.
5. Python absorbs standardized paths.
6. Claude remains on edge cases and governance.
7. Repeat.

# KPI Targets
- lower recurring inference cost
- increase Python-handled volume
- maintain or improve decision quality
- reduce exception rate over time
- shorten codification payback period

# Canonical Outcome
The target end-state is **maximum profitable automation with controlled reasoning spend**.
