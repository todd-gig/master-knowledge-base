# Objective
Measure the business delta between Claude and Python execution paths.

# Benchmark Dimensions
- unit cost,
- latency,
- decision quality,
- agreement rate,
- exception rate,
- human review burden,
- codification savings potential.

# Fixtures
Use stable benchmark fixtures with:
- shared input payloads,
- expected classifications,
- expected architecture,
- expected quality thresholds.

# Decision Rule
Benchmark results should drive:
- routing policy updates,
- codification priority,
- prompt release approvals,
- ROI comparison between execution modes.
