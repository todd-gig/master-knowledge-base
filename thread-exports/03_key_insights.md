# Key Insights

## Major insights

- The project treats enterprise design as a **capability architecture** rather than a headcount chart.
- Skills must be decomposed into atomic domains so they can map to roles, certifications, and negotiation activities.
- Negotiation is a cross-functional execution layer that must be staffed intentionally.
- Human supervision is a first-class system requirement.
- Claude projects become more reliable when instructions are split into:
  - root logic
  - skill ontology
  - team-structure rules
  - negotiation framework
  - output template
  - routing index

## Reusable heuristics

- Do not assume the answer is more people.
- Separate **owner**, **supporting functions**, and **supervision requirements**.
- Preserve explicit human review for sensitive decisions.
- Use the fewest roles necessary, but do not under-assign where risk increases.
- Load specialized companion files only when the task trigger requires them.
