# Requirements

## Functional requirements

- Generate modular Markdown files with YAML frontmatter.
- Produce Obsidian-ready structure.
- Create Claude-compatible prompt files.
- Include a routing index for companion files.
- Package outputs in downloadable zip bundles.
- Support skill assignment, team design, and negotiation activity generation.

## Non-functional requirements

- human-readable
- machine-usable
- modular
- reusable
- high signal density
- future-automation friendly

## Governance requirements

- preserve human accountability
- identify approval layers
- make AI supervision explicit
- support certification/readiness logic
